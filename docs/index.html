📍 Nearby Saved
How it works Download GitHub

Get notified when you're near any place you saved in Google Maps.

Google Maps lets you save places — but never tells you when you walk past one. Nearby Saved does, and it scales to thousands of saved places by using a geohash spatial index instead of iOS's limited geofences. Private, on-device, and battery-efficient.

⬇︎ Download the build View source on GitHub

What it does

Everything runs locally on your iPhone. Your location never leaves the device.

🧭

Proximity alerts

Notifies you when you enter the radius of a saved place — 250 m to 1 km, your choice.

🗂️

Thousands of places

A geohash spatial index keeps nearby searches under 100 ms even with 10,000+ places.

🔋

Battery-friendly

Significant-change + visit monitoring; accuracy rises only when a place is close.

🚗

Adaptive radius

Wider alerts when driving, tighter when walking — auto-detected from your speed.

🔕

Smart de-duplication

Alerts on entry only; re-notifies after you leave and return, with a cooldown.

📦

Batching

“You have 5 saved places nearby” instead of five separate buzzes.

🏷️

Category filters

Enable coffee & restaurants, mute hotels & museums — perfect for travelling.

🔒

Private by design

On-device SwiftData storage. No account, no cloud, offline capable.

How it works

Instead of one geofence per place (iOS caps that at ~20), it queries a spatial index.

   Import Google Takeout saved places
                 │
        Build geohash spatial index
                 │
   Significant location change / visit
                 │
   Query index → ~9–25 geohash cells only
                 │
   Precise Haversine on the few candidates
                 │
   Filter (category · transport · time · cooldown)
                 │
        Local notification  →  record to prevent duplicates

Download

The build is compiled automatically on a macOS runner by GitHub Actions.

📲 Compiled app (iOS Simulator)

A ready-to-run simulator build of the app, produced by CI.

⬇︎ Get the latest release

Run it with:

unzip SavedPlacesNotifier-simulator.zip
xcrun simctl boot "iPhone 16"
xcrun simctl install booted SavedPlacesNotifier.app
xcrun simctl launch booted com.bilalalhmoud.savedplacesnotifier

🛠️ Build for your iPhone

To install on a real device you sign it with your own Apple ID in Xcode (an Apple requirement — unsigned builds can't run on hardware).

git clone https://github.com/bilalalhmoud/saved-places-notifier
cd saved-places-notifier
brew install xcodegen
xcodegen generate
open SavedPlacesNotifier.xcodeproj
# Select your team, pick your iPhone, press ▶

Why no “install on iPhone” button? iOS apps can only be installed via the App Store, TestFlight, or a signed build from Xcode. A native app cannot run inside a web browser, so this page offers the CI-compiled build and one-command instructions to run or sign it yourself.

Import your places

From a Google Takeout export.

  • Open Google Takeout → deselect all → choose Saved (and/or Maps “your places”).
  • Download and unzip the archive.
  • In the app: Home ▸ Import and pick the .geojson / .json / .csv files. Each file name becomes a list/category.

Under the hood

Correctness-critical logic lives in a pure Swift package, unit-tested in CI without a simulator.

SwiftUI + SwiftData

Modern declarative UI and on-device persistence.

Core Location

Significant-change & visit monitoring for low-power background awareness.

Geohash index

Bucketed cells + neighbour grid for O(candidates) proximity search.

Tested engine

Haversine, importer, dedup & cooldown covered by XCTest suites.
